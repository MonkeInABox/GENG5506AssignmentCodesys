Execute State Fault Code Explanation
1. Purpose
This document explains when each FaultCode appears during the Execute state of the filling system. The codes are
grouped by PackML response type: Hold, Suspend, and Abort. The goal is to make the fault handling easier to test and
easier to describe in the report.
2. State Classification

The machine stops safely because the
problem is more serious. The system
must go through Aborting, Aborted,
Clearing, and then restart from Stopped.

4. Detailed Explanation by Response Type
4.1 Hold faults
FaultCode 50 and FaultCode 60 are Hold conditions. They are caused by low fluid level during the filling process. The
machine should pause, keep the process step, and wait for the operator to refill the tank. After the low fluid input is
cleared, UnholdCmd can be used to return to Execute.

4.2 Suspend faults
FaultCode 30 and FaultCode 40 are Suspend conditions. They represent temporary upstream or downstream flow
interruptions. The machine stops safely and waits until the external condition is corrected. Then UnsuspendCmd can
return the system to Execute.

4.3 Abort faults
Abort faults are serious problems. When one of these faults appears, the machine should stop safely, raise the alarm,
and go to Aborting and then Aborted. The operator should clear the issue before using ClearCmd.

6. Notes for the Report
• FaultCode 50 and 60 are used for internal pause conditions, so they are classified as Hold.
• FaultCode 30 and 40 are used for temporary infeed or outfeed flow problems, so they are classified as Suspend.
• Timeouts, conflicting sensor results, invalid container type, conveyor fault, and unknown ExecStep are classified as
Abort.
• During Hold or Suspend, Execute timers should be disabled so that a temporary pause does not become a timeout
fault.
• The optional signals Valve_A_FB, Valve_B_FB, and Zero_Speed_SW can be reserved for future feedback
monitoring. They are not required in the current simplified Execute fault logic.
